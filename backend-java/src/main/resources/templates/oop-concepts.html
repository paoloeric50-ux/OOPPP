OOP Concepts

Explore how Object-Oriented Programming principles are applied in this payroll system.

🔒
Encapsulation

Data and methods are bundled together within classes, hiding implementation details from the outside world.

public External API methods
protected Shared within hierarchy
private Internal implementation
🧬
Inheritance

Employee subclasses inherit common attributes and behaviors from the abstract base Employee class.

Employee (abstract)
└─ FullTimeEmployee
└─ PartTimeEmployee
└─ ContractEmployee
🔄
Polymorphism

Each employee type implements computeSalary() differently — the same method name produces different calculations.

🟣 FullTime: monthly × days
🟡 PartTime: hours × hourly rate
🩷 Contract: days × daily rate
🏗️
Abstraction

Abstract Employee class defines the interface without specifying implementation — only what must be done, not how.

abstractcomputeSalary()
abstractgetSalaryBreakdown()
interfaceEmployee contract
